About the Telco Cloud Platform Upgrade Guide
   VMware Telco Cloud Platform™ is a cloud-native platform powered by field-proven VMware Telco Cloud
   Infrastructure™ coupled with automation. It provides a cloud-first approach that delivers operational agility
   for virtual, cloud-native 5G, and Edge network functions. Telco Cloud Platform helps Communication
   Service Providers (CSPs) roll out 5G networks and deploy 5G applications, solutions, and services more
   quickly across the multi-cloud infrastructure.
   This Platform Upgrade Guide describes the process of upgrading Telco Cloud Platform to release 5.0.2
   based on Telco Cloud Platform Reference Architecture. Before upgrading the Telco Cloud solution
   components, ensure that you are knowledgeable on the following:
   •   Telco Cloud Platform architectural concepts and deployment scenarios. For more information, see the
       VMware Telco Cloud Reference Architecture Guide.
   •   Telco Cloud Platform design considerations. For more information, see the VMware Telco Cloud
       Planning and Design Guide.
   •   Upgrade procedures of the Telco Cloud Platform components. For more information, see the
       documentation of each component.
   Intended Audience
   This guide is for telecommunications and solution architects, sales engineers, field consultants, advanced
   services specialists, and customers who want to upgrade Telco Cloud Platform.


Telco Cloud Platform Architecture
   VMware Telco Cloud Platform combines VMware Telco Cloud Infrastructure and VMware Telco Cloud
   Automation™. VMware Telco Cloud Infrastructure is an evolution of VMware vCloud® NFV™. VMware
   Telco Cloud Automation contains multi-domain orchestration and automation capability. VMware Telco
   Cloud Platform also embeds Tanzu Kubernetes Grid that allows CSPs to build, manage, and run
   containerized workloads across private, telco, edge, and public clouds.
   To manage massive data traffic, 5G is designed for a distributed architecture. The user plane and control
   plane are separated, and the data center and network infrastructure are based on hierarchical multi-tiered
   layers. For more information about each layer, see the Telco Cloud Reference Architecture Guide. For
   design considerations of components in each layer, see Telco Cloud Planning and Design Guide.

Upgrade Prerequisites
    Ensure that the following prerequisites are met before upgrading VMware Telco Cloud Platform:
    •   Upgrade Checklist
    •   Snapshot and Backup Requirements

Telco Cloud Platform Upgrade Checklist
    VMware Telco Cloud Platform follows strict product upgrade guidelines. You must adhere to the upgrade
    checklist before performing the upgrade.


                                                                                                                     244
   1. Ensure that the existing Telco Cloud solution deployed in your production environment is functional.
   2. Ensure that the Telco Cloud Platform 5.0.2 bundle is downloaded and accessible.
   3. Ensure that the production environment has enough storage capacity for backups and snapshots.
   4. Take a snapshot and back up the management components of the Telco Cloud solution in the
      production environment.
   5. Ensure that all the components of the Telco Cloud solution in the production environment are patched
      to the latest.
   6. Based on the components identified in the Upgrade Sequence section, develop an upgrade plan for the
      Telco Cloud solution.

Snapshot and Backup Requirements
   Before you upgrade the Telco Cloud solution components, ensure that you take snapshots and backups of
   all the components as a prerequisite. If a component upgrade fails, you can use the snapshot and backup
   to restore the component to its previous version.
   •   VMware Cloud Director (cells and database):
       1. Take a snapshot of the VMware Cloud Director appliance. For instructions, see Creating a Snapshot
          of a VM.
       2. Back up the embedded database of the VMware Cloud Director appliance. For instructions, see
          Backup and Restore of VMware Cloud Director Appliance.
   •   vSphere Replication: Take a snapshot of the vSphere Replication Appliance VM. For instructions, see
       Creating a Snapshot of a VM.
   •   Live Site Recovery: Take a snapshot of the Live Site Recovery virtual appliance and Back Up the
       Embedded vPostgres Database. For instructions, see Creating a Snapshot of a VM and Back Up the
       Embedded Database.
   •   vCenter:
       •  Take a VM snapshot of the vCenter appliance. For instructions, see Creating a Snapshot of a VM.
       •  Create a file-based backup of the vCenter appliance using the vCenter interface. For instructions,
          see File-Based Backup and Restore of vCenter Server.
   •   NSX Manager: Back up the NSX Manager configuration. For instructions, see Backing Up and
       Restoring NSX Manager or Global Manager.
   •   ESXi and vSAN: Back up the ESXi host configuration. For instructions, see KB2042141.
   •   Aria Operations for Networks: Take a VM snapshot of the Aria Operations for Networks deployment.
       For instructions, see KB55829 and Backup a VMware Aria Operations for Networks Deployment.
   •   Aria Automation Orchestrator: Take a VM snapshot of the Aria Automation Orchestrator deployment.
       For instructions, see Creating a Snapshot of a VM.
              CAUTION:
               Aria Automation Orchestrator 8.x does not support memory snapshots. Before taking the
               snapshot, ensure that theSnapshot the virtual machine's memory option is deactivated.
   •   Aria Operations for Logs: Take a VM snapshot of the Aria Operations for Logs cluster nodes. For
       instructions, see Creating a Snapshot of a VM.
   •   Aria Operations: Take a VM snapshot of the Aria Operations nodes. For instructions, see Creating a
       Snapshot of a VM.
   •   Telco Cloud Automation: Take a VM snapshot of the Telco Cloud Automation appliance. For
       instructions, see Creating a Snapshot of a VM.
   •   Avi Load Balancer: Take a VM snapshot of Avi Load Balancer. For instructions, see Creating a
       Snapshot of a VM.

Upgrading Telco Cloud Platform
    This section outlines the high-level steps to upgrade the required and optional components of VMware
    Telco Cloud Platform. For detailed instructions, refer to the corresponding component documentation.


Upgrade VMware Cloud Director
    This section describes how to upgrade VMware Cloud Director to 10.6.1.
    •   Verify that you have superuser credentials for the VMware Cloud Director Cells in your VMware Cloud
        Director server group.
    •   Verify that all ESXi hosts are enabled. Starting with VMware Cloud Director 10.3.2a, deactivated ESXi
        hosts are not supported.
    •   Back up the embedded database of the VMware Cloud Director appliance. For instructions, see Backup
        and Restore of VMware Cloud Director Appliance.
    •   Take a snapshot of the primary VMware Cloud Director appliance. For instructions, see Creating a
        Snapshot of a VM.
    1. Upgrade the VMware Cloud Director appliance to 10.6.1.
        If the vCenter gets disconnected after upgrading VMware Cloud Director, reconnect vCenter from the UI
        as follows:
        a. Navigate to Resources > Infrastructure Resources > vCenter Server Instances.
        b. Select the vCenter Instance.
        c. Click Reconnect.
       For more information, see KB78885.
    2. If Avi Load Balancer (formerly NSX Advanced Load Balancer) was deployed as part of VMware Cloud
       Director, upgrade Avi Load Balancer.

Upgrade vSphere Replication
    To upgrade vSphere Replication, you must upgrade the required components in your vSphere Replication
    environment.
    Take a snapshot of the vSphere Replication Appliance VM. For instructions, see Creating a Snapshot of a
    VM.
            CAUTION:
            •   Before upgrading the components on the recovery site, you must upgrade all components on the
                protected site.
            •   During the upgrade, if there are issues that prevent you from accessing the protected site, use
                the recovery site to restore the protected site to its original state.
    Upgrade vSphere Replication to 9.0.2.

Upgrade Live Site Recovery
    When you upgrade VMware Live Site Recovery you must complete all the upgrade tasks on the protected
    site first and then start the upgrade tasks on the recovery site.
    •   Take a snapshot of the Live Site Recovery virtual appliance. For instructions, see Creating a Snapshot
        of a VM.
    •   Follow the Prerequisites and Best Practices for Live Site Recovery Upgrade.
    •   Ensure that vSphere Replication is upgraded to version 9.0.2.
    Note:

Upgrading Telco Cloud Automation
   This section describes how to upgrade VMware Telco Cloud Automation to 3.3.0.1.
   Telco Cloud Platform 5.0.2 supports the following Telco Cloud Automation upgrade paths:
   •   Telco Cloud Automation 3.3 > 3.3.0.1
   •   Telco Cloud Automation 3.2 > 3.3.0.1
   •   Telco Cloud Automation 3.1.1 > 3.3.0.1
   •   Telco Cloud Automation 3.1 > 3.3.0.1
   •   Telco Cloud Automation 2.3 > 3.3.0.1

   Upgrade Telco Cloud Automation from Release 3.1 or Later to 3.3.0.1
   This section describes how to upgrade VMware Telco Cloud Automation from release 3.1 or later to
   3.3.0.1.
   •   Ensure that components of Telco Cloud Automation 3.1 or later are activated.
   •   Back up the Telco Cloud Automation Manager and Telco Cloud Automation Control Plane appliances.
       For instructions, see Backing Up and Restoring the System in the Telco Cloud Automation Deployment
       Guide.
   •   Download the Telco Cloud Automation 3.3.0.1 Upgrade bundle from the Telco Cloud Platform 5.0.2
       Advanced downloads page.
   1. If you have an existing Airgap server in Telco Cloud Automation 3.1 or later, upgrade the Airgap server
      to Telco Cloud Automation 3.3.0.1. For instructions, see Upgrade Airgap Appliance.
   2. Upgrade the Telco Cloud Automation Manager and Telco Cloud Automation Control Plane appliances.
      For instructions, see Upgrading VMware Telco Cloud Automation.
   1. Verify that you have successfully upgraded Telco Cloud Automation:
       a. Open the Telco Cloud Automation web interface.
       b. Verify that the dashboard displays the upgraded version number of Telco Cloud Automation.
       c. Verify that all the services are up and running by logging in to one of the following URLs:
        • https://<TCA Manager_FQDN>:9443
        • https://<TCA Control Plane_FQDN>:9443
   2. Update the existing Harbor with ca.crt under TCA Manager > Infrastructure > Partner Systems.
       Note:
      Applies only to Harbor that is using a private CA-signed or self-signed certificate.
   3. Verify and fix the alerts displayed on the TCA-M dashboard.

   deployments. You can deploy and upgrade Harbor as a CNF through Telco Cloud Automation.
   To download Harbor for CNFs, see Drivers & Tools on the Telco Cloud Platform 5.0.2 Advanced
   downloads page.
